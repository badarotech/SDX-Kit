Autoria guiada de **tasks** de uma jornada. Leia `constitution.md` e o `needs.md` da jornada.
É a versão manual do `/specify`.

Conduza com perguntas:

1. Qual jornada? Liste as necessidades disponíveis (`NEED-xx`) e pergunte qual atender.
2. Para a need escolhida, derive uma ou mais tasks:
   - descrição executável (pequena o bastante para desenhar e prototipar);
   - critérios de aceite herdados ou refinados da need;
   - dependências entre tasks;
   - se a task ficar grande demais, sugira dividir.
3. Atribua `TASK-xxx`, referencie a need de origem, escreva/atualize `tasks.md`, atualize a
   cobertura (toda need com ao menos uma task), mostre resumo e pare. Sugira `/design`.

Regras: não avance para design ou protótipo aqui; se um critério não puder virar task verificável,
aponte o problema em vez de mascarar.
